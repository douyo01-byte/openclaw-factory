# OpenClaw Development Rules

## 会話の基本ルール

1. まず事実確認
2. 進んでいる方を正とする
3. まず開発部門重視
4. 外部課金は自動で行わない
5. 破壊的変更前には現役 / 保険 / 削除候補を分ける
6. 重くなったら引き継ぐ

## 出力ルール

- ターミナル貼り付け前提
- コマンド中心
- 説明は最低限
- コード内コメント不要
- 改行崩れに注意
- まずは壊さない

## コード方針

1. 1ファイル1責務を意識
2. Telegram受信
3. DB保存
4. ルーティング
5. 実行
6. 通知
を混ぜすぎない
7. print / log は最低限の状態遷移だけ出す
8. 例外は握りつぶしすぎない
9. ただし loop 全停止は避ける
10. main() を持たせる
11. 単体実行できる形を残す
12. env 未読込時に死因が分かるようにする
13. status / processed / applied_at の整合を崩さない
14. ソースがあるなら pyc 依存にしない
15. rename より wrapper で段階移行した方が安全
