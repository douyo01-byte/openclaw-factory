In tick() the code always updates notified_at=datetime('now') even when tg_send() failed and returned an empty string, causing failed sends to be treated as delivered and skipped on future runs. Change the update logic so that notified_at is set only when tg_send() returns a non-empty message id. On failure, set notified_msg_id=NULL (or leave it NULL) so the row remains eligible for retry. This is a small, low-risk change localized to the tick() update statements.
