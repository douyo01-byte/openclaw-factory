Introduce a small module-level TTL cache (e.g. dict mapping path -> (timestamp, json)) and check it at the start of gh_api(path). If a fresh cached response exists (TTL ~30s), return it without issuing an HTTP request. On successful HTTP 2xx responses, store the parsed JSON in the cache with current timestamp. This avoids repeated requests for the same PR within short intervals during a single watcher loop, lowering API usage and transient rate-limit exposure. Implementation is a few lines: add GH_CACHE and GH_CACHE_TTL constants, check/return cached payload at top of gh_api, and save into cache just before returning r.json(). Keep behavior unchanged on errors (do not cache errors).
