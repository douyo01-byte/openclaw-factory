Replace per-row c.execute(...); c.commit() inside the loop with collecting matching row ids and executing a single parameterized UPDATE that sets processed=1,status='skipped',applied_at=? WHERE id IN (?,?,...); then call c.commit() once. This keeps the existing behavior and limit(50) but reduces the number of transactions, WAL writes, and lock contention, improving throughput and lowering chance of busy/timeout errors. Implementation is low-risk: generate the IN-clause placeholders from the collected ids, skip doing an UPDATE if no ids matched, and reuse now() for the applied_at timestamp.
