Currently strip_role_words does simple str.replace for each alias, which leaves Latin aliases if they appear in different cases (e.g. 'Scout', 'SCOUT') and can leave awkward spacing. Change strip_role_words to: 1) perform regex replacements using re.sub(re.escape(name), '', text, flags=re.IGNORECASE) for each alias so Latin/ASCII variants are removed regardless of case; 2) collapse leftover whitespace and run _normalize_text (already in this file) before returning. This is a small, local refactor that preserves behavior for Japanese aliases while making alias removal robust and producing a clean query string for downstream handlers.
