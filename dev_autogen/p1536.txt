Replace direct int(...) conversions for SLEEP_SEC and FAIL_NOTIFY_THRESHOLD with a small helper that catches ValueError and falls back to a safe default while emitting a warning. This prevents the module from raising an exception during import when an environment variable is set to a non-numeric value (which would crash the daemon). Implementing this is a minimal change: add a parse_int_env(name, default) function, call it for OCLAW_POLL_INTERVAL_SEC and OCLAW_POLL_FAIL_NOTIFY_THRESHOLD, and print/log a short warning when fallback occurs.
