In llm(), add a short requests timeout (e.g. timeout=15) to the requests.post call and call r.raise_for_status() before r.json(). This prevents the process from hanging indefinitely if the OpenAI API is slow or unresponsive and surfaces non-200 HTTP errors as exceptions immediately. Expected effect: reduces risk of stuck CI/agents, fails fast on upstream API errors so callers can handle/retry, and improves overall reliability with minimal code change.
