Wrap the requests.post send in a small retry loop (max 3 attempts) that catches requests.exceptions.RequestException and treats HTTP 5xx and 429 as transient. On transient failure retry with exponential backoff (e.g. 1s, 2s, 4s). Only mark the row as sent and update sent_at when a send succeeds; if all attempts fail, log/print the failure and continue to the next row without touching the DB. This is a small, low-risk change limited to the per-row send logic that reduces lost/partial runs due to transient network or API throttling errors.
