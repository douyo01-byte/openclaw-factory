In the current loop, when chat_main() raises an exception the code calls time.sleep(max(INTERVAL, 5)) inside the except block and then unconditionally calls time.sleep(INTERVAL) after the try/except, causing a doubled sleep on errors. Change: add a 'continue' immediately after the sleep() in the except block (or set a single next_sleep variable and sleep once after the try/except). Expected effect: removes unintended extra delay after failures so retries and failure-notify timing behave as intended, improving responsiveness and reliability with a minimal, low-risk one-line change.
