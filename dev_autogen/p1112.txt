In main() the code attempts to extract the PR number from the GH CLI output using: m = re.search(r"/pull/(\\d+)", pr_url). The pattern incorrectly looks for a literal backslash and therefore fails to match common PR URLs. Change the line to use a proper digit capture: m = re.search(r"/pull/(\d+)", pr_url). Expected effect: reliably parse PR numbers from returned URLs so pr_num is set (not None), which prevents missing pr_number in DB updates and downstream records, fixes inaccurate state tracking, and makes subsequent automation/notifications that rely on pr_number work correctly. This is a minimal, low-risk one-line fix limited to this file.
