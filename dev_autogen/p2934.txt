In refine_once(), wrap the work for each proposal (building prompt, calling ask_llm, and performing DB updates/inserts) in a try/except so one failing proposal does not abort the entire batch. On exception: (1) insert a proposal_conversation row for that proposal with role='assistant' and message='[spec_refiner_v2] error: <short error message>', truncating the error text to e.g. 500 chars; (2) upsert proposal_state to set stage='error' (or 'refine_error') and updated_at to now; (3) print the error to stdout as currently done; then continue to the next proposal. This is a small, low-risk change localized inside refine_once() that prevents a single LLM/API/DB error from stopping processing of other proposals and provides an in-DB diagnostic trail.
