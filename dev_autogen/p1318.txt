The code currently uses re.search(r"/pull/(\\d+)", pr_url) which looks for a literal backslash and therefore fails to extract numeric PR IDs from gh output. Change the pattern to re.search(r"/pull/(\d+)", pr_url). This is a single-line, low-risk bugfix: it will allow pr_num to be populated with the actual integer ID, so the dev_proposals.pr_number and downstream ceo_hub_events will be set correctly and tooling that relies on numeric PR IDs will stop seeing null values.
