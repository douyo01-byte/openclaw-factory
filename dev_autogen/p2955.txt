Wrap the HTTP call in ask_llm with a small retry loop that handles transient network/timeouts and 429/5xx responses. On requests.exceptions.RequestException or response.status_code in (429, 500..599) retry up to 4 attempts with exponential backoff and jitter (e.g. 1s, 2s, 4s + small random jitter). Log each retry attempt with attempt number, status code or exception, and sleep duration using existing print(..., flush=True). On final failure raise the original exception. This is a small, low-risk change contained to ask_llm and avoids spamming the LLM on obvious transient failures while preserving existing behavior on persistent errors.
