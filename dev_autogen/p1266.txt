Before calling sqlite3.connect(DB) create the parent directory for DB if it doesn't exist (e.g. with: d=os.path.dirname(DB); if d: os.makedirs(d, exist_ok=True)). This prevents the script from failing with "No such file or directory" when DB_PATH points to a file inside a non-existent directory (the current default 'data/openclaw.db' requires 'data' to exist). Expected effect: eliminates startup failures on fresh deployments, ensures the database file can be created, and is a low-risk change that does not alter runtime behavior when the directory already exists.
