When building the message text from title+body, ensure it does not exceed Telegram's 4096 character limit. Implement a simple guard that truncates the composed text to 4000 characters (leaving space for an appended '… (truncated)' suffix) and append that suffix. Also print a short warning with the row id when truncation occurs so operators can notice truncated notifications. This is a small, low-risk change limited to the text construction and logging and prevents sendMessage HTTP 400 errors for overly long DB rows.
