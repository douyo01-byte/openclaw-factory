Add low-risk, in-file logging in tg_send to improve observability of why notifications fail. Concretely: print a clear warning when BOT_TOKEN or CHAT_ID are not set; catch requests exceptions and print the exception repr with a short hex/timestamp context; when the HTTP response status is not 200 print status_code and the first ~500 chars of response.text; when Telegram's JSON 'ok' is false print the parsed JSON (truncated) and the truncated message payload that was attempted. Keep all logs concise and truncated to avoid leaking large payloads. No behavior change to return values: tg_send still returns message_id or empty string.
