Wrap the daemon and factory sqlite3 connections in try/finally (or use context managers) so both connections are closed even if an exception occurs. Concretely: after creating d = sqlite3.connect(DAEMON_DB) and f = sqlite3.connect(FACTORY_DB), move the main work into nested try blocks and call f.close() and d.close() in finally blocks. This prevents leaked DB handles and persistent WAL locks when the script exits due to network or JSON errors.
