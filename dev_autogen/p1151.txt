Modify scan_secrets to avoid opening and scanning very large or binary files: add MAX_SECRET_SCAN_BYTES = int(os.environ.get('OCLAW_MAX_SECRET_SCAN_BYTES', '200000')) at top, and in scan_secrets check os.path.getsize(p) and skip files larger than the limit. Additionally, open only the first 4096 bytes to detect null bytes (b'\x00') and skip files that look binary. Only read the full file when size <= limit and no null bytes are found. Expected effect: reduces memory and I/O when running on repos with large artifacts, prevents long hangs or high memory usage from scanning large binaries, and reduces false positives and CPU wasted on irrelevant files. This is a small, low-risk guard change localized to scan_secrets.
