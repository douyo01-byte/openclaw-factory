In _normalize_text the variable jp is set as a raw string (r"\u3040-..."), which preserves the backslash-u escapes instead of producing the intended Unicode characters/ranges. As a result the regex (?<=[{jp}])\s+(?=[{jp}]) does not correctly detect Japanese characters and fails to remove spaces between them. Change the jp assignment to a normal string: jp = "\u3040-\u30ff\u4e00-\u9fff". Expected effect: the inter-Japanese-character whitespace removal will work correctly, improving normalization (and downstream title extraction / item resolution) with a single low-risk one-line change.
