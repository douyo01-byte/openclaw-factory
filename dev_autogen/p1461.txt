Add lightweight error handling so a single transient failure (network, HTTP error, or invalid JSON) doesn't crash the whole run or leave rows stuck as 'new'. (2) In send(): catch requests exceptions and log/return failure (no crash). (3) In main(): wrap per-row processing in try/except so any exception updates the processed inbox_commands row to status='failed' with an error message (via an update query) and continues to the next row. These are small, low-risk edits limited to this file that improve reliability and ensure the loop continues on individual failures.
