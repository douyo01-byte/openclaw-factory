Currently the script uses random.* to synthesize price/units which makes runs non-deterministic and harder to test. Add an optional, low-risk seeding step at top: read AUTO_PLAN_SEED from the environment and, if set, call random.seed(int(seed)). This is a minimal change (add import os and a couple lines) that preserves existing behavior when the env var is absent but enables reproducible runs for tests or CI. Keep all DB logic unchanged.
