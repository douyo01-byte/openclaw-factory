Create a small persistent state row in the existing SQLite DB to store the last processed Telegram update_id and restore it on startup. Concretely: at startup execute `CREATE TABLE IF NOT EXISTS ingest_state(key TEXT PRIMARY KEY, value INTEGER)` and then `SELECT value FROM ingest_state WHERE key='last_offset'` to set the in-memory `offset` variable (default 0). After successfully handling each update (inside the for-loop, after setting `offset = u['update_id']` and inserting the message), write the new offset with `INSERT OR REPLACE INTO ingest_state(key,value) VALUES('last_offset', ?)`. This is a low-risk change (DDL is idempotent, single-row upsert) that prevents re-processing of already-handled messages after process restarts and reduces duplicate inserts/errors.
